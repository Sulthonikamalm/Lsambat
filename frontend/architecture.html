SurabayaSambat v2 — System Architecture

Arsitektur Lengkap Sistem Incremental Comment Monitoring

1. State Machine — Mode Sistem
IDLE
Belum ada baseline
Klik Initialize Baseline
INITIALIZING
Scraping semua komentar lama
Selesai (otomatis)
WARM-UP MODE
Stabilisasi baseline — komentar yang ditemukan TIDAK masuk dataset
Klik Mulai Monitoring
OFFICIAL MONITORING
Komentar baru MASUK dataset (new_comments.csv)
Klik Stop
STOPPED
Dataset final siap download
Reset tersedia di semua mode → kembali ke IDLE
2. Alur Lengkap — Initialize Baseline
User klik "Initialize Baseline" Di browser dashboard (http://localhost:5000)
POST /api/initialize
Browser kirim HTTP POST ke Flask Backend via fetch() dari app.js
Flask: app.py menerima request Panggil monitor.initialize_baseline(limit=1000)
monitor.py → scraper.scrape_comments() Pilih token (round-robin), buat actor input JSON
HTTP POST + JSON payload
Apify REST API v2 POST /v2/acts/apify~instagram-scraper/run-sync-get-dataset-items
Apify jalankan actor di cloud
Instagram Scraper Actor berjalan Scrape komentar dari post URL di server Apify (~10-30 detik)
Return JSON array of comments
scraper.py: Filter error items Buang item dengan "error":"no_items", return komentar valid saja
monitor.py: Hash setiap comment ID SHA-256("instagram|" + comment_id)[:16]
Simpan SEMUA ke seen_comments.json source: "baseline", is_baseline: true
TIDAK masuk new_comments.csv
Mode berubah: IDLE → WARM-UP Emit WebSocket "status_change" + "log" ke dashboard
Dashboard update: "Baseline: N komentar tercatat" Tombol Stabilkan & Mulai Monitoring aktif
3. Alur Lengkap — Monitoring Cycle (Deteksi Komentar Baru)
User klik "Jalankan Sekarang" atau Timer 3 menit trigger
POST /api/run-once
Flask: Cek mode saat ini Warm-up? → run_stabilize() | Official? → run_monitoring_cycle()
monitor.py → scraper.scrape_comments(limit=50) Ambil komentar terbaru saja (hemat CU)
HTTP POST ke Apify API
Apify jalankan actor → return ~15 komentar terbaru Free tier: max ~15 | Paid tier: sesuai limit
JSON response
Untuk SETIAP komentar: hitung hash → cek seen_comments.json
Apakah comment_hash sudah ada di seen_ids?
SUDAH ADA
SKIP (duplikat dicegah) duplicates_prevented += 1
BELUM ADA (BARU!)
Mode saat ini?
WARM-UP
Tambah ke seen_ids source: "warmup"
is_baseline: true
TIDAK masuk CSV
OFFICIAL
Tambah ke seen_ids source: "official"
is_baseline: false
MASUK new_comments.csv
WebSocket emit Dashboard tampilkan komentar baru
Log siklus ke monitoring_log.csv cycle, scraped, new_found, duplicates, status
Dashboard update: analytics, chart, tabel komentar Countdown reset → tunggu 3 menit → ulang siklus
